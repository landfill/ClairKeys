## Why this exists

The user reported low notes on the deployed site sounding like bass noise rather than piano tone.
Every note was synthesised as a single `sine` oscillator behind a lowpass at `frequency * 4`.

**The filter was inert.** A sine's spectrum is one line, so there was never anything for the lowpass
to remove — it read in the source as a timbre-shaping stage while doing nothing at all.

The defect is inaudible in the treble, where a bare sine still reads as a pitch, which is why the
report is specifically about low notes. A piano's bass strings are shorter than their pitch requires,
so their fundamental is weak and the ear infers the note from the partials. At A0's 27.5 Hz a lone
sine leaves nothing to infer from, and small speakers reproduce almost none of that fundamental.

## Measured change

| Note | partials before → after | cutoff before → after | fundamental share |
|---|---|---|---|
| A0 (27.5 Hz) | 1 → **24** | 110 Hz → **2600 Hz** | 17.5% |
| C2 (65.4 Hz) | 1 → 24 | 262 Hz → 2600 Hz | 30.0% |
| C4 (261.6 Hz) | 1 → 24 | 1047 Hz → 3140 Hz | 51.5% |
| C8 (4186 Hz) | 1 → 3 | 16744 Hz → 16000 Hz | 79.3% |

The C8 partial count is limited by `MAX_PARTIAL_HZ` (16 kHz), which sits below Nyquist at 44.1 kHz so
no partial can fold back as an alias.

The envelope changed for the same class of reason as the filter: it held `sustain 0.6` for the note's
entire length, which is a sustained instrument. A struck string only loses energy after the hammer,
so it now decays exponentially toward 18% of peak across the note.

## Design decisions

- **Pure module, not inline.** Fused with `createOscillator` these decisions cannot be tested, which
  is why a filter that did nothing survived unremarked. Same seam `audioScheduler` used for window
  selection — the one that made issue #18's 10-second cap visible.
- **One `PeriodicWave`, not one oscillator per partial.** `VOICE_LIMIT` counts voices, so stacking 24
  oscillators for a bass note would silently cut effective polyphony to a fraction. Rejected on that
  ground despite being simpler to read.
- **`exponentialRampToValueAtTime`, not `setTargetAtTime`.** The latter never exactly reaches its
  target, so the release would begin from an unknown level. The ramp target is floored at `1e-4`
  because `exponentialRamp` cannot pass through or land on zero.
- **Rejected: Tone.js Sampler with real piano samples.** Two Tone.js synthesis paths
  (`src/services/audioService.ts`, `src/lib/audio/piano.ts`) already sit unused alongside this one.
  Adding sample assets and a third path is a P2-A architecture decision, not a timbre fix.
- **`PeriodicWave` construction is wrapped in try/catch.** Test doubles and older engines may not
  implement it; the fallback is `type = 'sine'`, which is the pre-existing timbre rather than a new
  failure mode.

## Two test assertions were wrong before the implementation was

Worth recording, since both looked reasonable:

1. The cutoff test demanded `f0 * MIN_KEPT_PARTIALS` for every key. At C8 that is 50 kHz — headroom
   above partials that cannot exist, since the aliasing ceiling leaves only 3. Narrowed to
   `min(MIN_KEPT_PARTIALS, actual partial count)`.
2. It then demanded the cutoff always exceed the old `4 * f0`. In the top octave `4 * f0` is itself
   past the aliasing ceiling, so the comparison is meaningless there. Scoped to notes where a 4th
   partial can exist.

Neither was an implementation fault. They are recorded because "the test failed, so change the code"
would have produced a worse module in both cases.

## Not verified — the important part

**Nobody has heard this.** jsdom has no Web Audio implementation and no offline renderer is installed,
so no waveform was rendered and no spectrum measured from actual output. Every number above describes
the coefficients handed to `PeriodicWave`, not what a browser produces from them.

Whether the result sounds like a piano is a listening judgement, and it has not been made. The change
also affects the timbre of **every** note, not only the low ones that prompted it — if the treble
reads as too bright or too dull, the knob is `TREBLE_ROLLOFF`, not a per-register special case in the
hook.
