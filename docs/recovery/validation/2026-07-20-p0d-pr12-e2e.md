# P0-D PR #12 E2E Gate Repair

Date: 2026-07-20 KST
Branch: `fix/lint-gate`

## Failure evidence

- `npx playwright test --list` discovered 50 tests: ten scenarios repeated across five browser/device projects.
- CI uses one worker and two retries, so missing-element waits are serialized and repeated.
- Every scenario began at `/dashboard` or `/sheet/:id`; `/dashboard` does not exist in `src/app`.
- The scenarios queried `data-testid` contracts that do not exist in application code.
- Both hosted E2E jobs therefore remained in the Playwright step while all prerequisite setup, migration, browser installation, and build steps had succeeded.

## Repair

- Deleted the aspirational authentication, dashboard, upload, and piano-player specifications that could not execute against the current application.
- Added a smoke suite using real public routes and accessible roles.
- Kept all five configured desktop/mobile browser projects.
- The suite verifies the real home response and navigation, the single main landmark, zoom-enabled viewport metadata, and public explorer navigation.

It passed all five zoom checks and exposed one deterministic strict-locator issue: both the hero and footer contain a public-explorer link. The navigation assertions now scope that link to the single `main` landmark.

The locator rerun passed all 15 E2E checks. The required `All Checks Complete` job also passed. A separate `PR Summary` comment job then received HTTP 403 because it declared no `issues: write` permission; that job now receives only the narrow permission required to publish its result comment.
